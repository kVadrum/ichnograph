## Picked (and why)
`<details>` + `<summary>` are the GitHub-flavored disclosure-widget
pair used in many open-source READMEs for collapsible sections (FAQs,
quick-start blocks, screenshots). Without them in the allowlist, a
README that wraps its description in a disclosure block surfaces the
literal `<details><summary>…</summary>…</details>` tags in
ichnograph's one-line summary — ugly and useless. `<figure>` and
`<figcaption>` ride along for the same reason: they show up around
captioned screenshots in READMEs and surface verbatim today. The
fix is a four-token addition to the existing allowlist regex in both
`readme.ts` and `notes.ts`, plus tests in both suites.

## Diff summary
A four-token extension to the existing curated HTML-tag allowlist in
both summary-extraction paths. The strip uses the same regex shape
already in production (`<\/?<tag>\b[^<>]*\/?>`), so behavior on every
existing wrapper is unchanged — wrapped content shines through, prose
placeholders like `<your-token>` remain literal because they don't
match any real tag. The reviewer's most useful checks: confirm the
disclosure-block test exercises both the opening and closing tag, and
that the multiline `<details>…</details>` paragraph still joins
correctly (existing paragraph-scan logic; not changed).
