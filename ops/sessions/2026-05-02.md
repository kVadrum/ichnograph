## Picked (and why)
Added reference-style link and image handling to both `stripMd`
(readme.ts) and `stripInlineMd` (notes.ts). The two helpers have been
evolving in lockstep across recent sessions (image syntax, italic,
strikethrough) so this fits the established pattern exactly. A common
README opening like `[![CI][ci-badge]][ci-url] A great tool…` previously
surfaced with brackets intact; now collapses to `CI A great tool…`.
Shortcut form `[text]` is intentionally untouched — too often literal in
prose (`[draft]`, `[WIP]`) to safely strip.

## Diff summary
Two new regex replacements per helper, sandwiched between the existing
inline-image and inline-link strips. The image-reference replacement
must run before the link-reference replacement for the same reason the
inline forms do — otherwise the link regex would consume the `[alt]`
part of an `![alt][ref]` and leave a stray `!`. Bare-shortcut form
(`[text]` with no following bracket or paren) is deliberately not
matched; a regex for that would mangle literal bracketed prose.
