## Picked (and why)
Autolink stripping. It's a clean, narrow, additive change in the same
groove as the last several sessions: one regex pair (URI form + email
form), zero collision risk with existing strippers, and obviously
useful — many READMEs include `<https://...>` for the homepage line
and `<email@x>` in a contact paragraph. Bare `<word>` literals in
prose (e.g. `Replace <name> with the project identifier`) stay intact
because the URI form requires a scheme and the email form requires
`user@host.tld`, and those tests pin the boundary.

## Diff summary
Adds two replace steps to the inline-markdown strip helper duplicated
in `notes.ts` and `readme.ts`. The first matches `<scheme:body>` where
scheme is `[a-zA-Z][a-zA-Z0-9+.-]{1,31}` (CommonMark requires a letter
start and 2–32 chars of restricted punctuation), and body is any
non-`<>\s`. The second matches `<user@host.tld>` requiring at least
one dot in the host portion to avoid swallowing `<x@y>` in prose. Both
forms drop only the angle brackets — the URL/email content is
preserved verbatim. Reviewer should sanity-check the email regex
boundary: `[^\s<>@.]+(?:\.[^\s<>@.]+)+` requires the host portion to
contain at least one dot, which rules out intranet-style hosts but
matches the common public form READMEs use.
