## Picked (and why)
Added pyproject.toml script detection to the entrypoints detector.
`[project.scripts]` (PEP 621) surfaces each name with a bare invoke
(scripts land on PATH after `pip install`), and `[tool.poetry.scripts]`
surfaces as `poetry run <name>` since that's the pre-install dev path.
The minimal TOML reader in `entrypoints.ts` only needs to pull flat
`name = "value"` pairs out of a known section, so no new dep and no
changes elsewhere.

## Diff summary
The new section reader finds `[section]` by literal string, slices the
body up to the next `\n[`, and extracts `name = "value"` pairs while
stripping `#` comments. It's deliberately narrower than the TOML reader
in `stack.ts` because console-script tables are flat; no multiline or
inline-table handling. Reviewer should sanity-check that a
`pyproject.toml` with only `[project]` (no scripts) still returns
`null` from `detectEntrypoints` — covered by the new test.
