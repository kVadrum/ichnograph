## Considered
- pubspec.yaml version extraction — Dart manifest had `version: null`
  hard-coded despite the field being standard in pubspec.
- gleam.toml version extraction — same shape, but Gleam in the wild is
  much rarer than Dart/Flutter; lower-impact pick.
- Cargo.toml workspace inheritance (`version.workspace = true`) — would
  require resolving the workspace root, more than one cycle's scope.
- mix.exs / Gemfile identity — these formats embed identity in `.mixs`
  attributes or `.gemspec` files, not the file ichnograph already reads;
  bigger change.

## Picked (and why)
pubspec.yaml version. Mirrors the deno.json work that just shipped
(2026-05-15) and the composer.json work before that — same shape:
the language is already detected, the manifest already read, just one
hard-coded `null` to replace with a regex pull. Small, isolated,
high-confidence. Dart is enough of a real ecosystem (Flutter especially)
that the field is broadly populated.

## Changes
- src/detect/stack.ts — extract top-level `version:` from pubspec.yaml
  with the regex `^version:\s*['"]?([^'"\s#]+)['"]?` so quoted values,
  bare values, Flutter build suffixes (`1.0.0+42`), and inline `#`
  comments are all handled. Anchored at column 0 so nested versions
  under `dependencies:` aren't matched.
- tests/stack.test.ts — 4 new cases covering bare value, Flutter build
  suffix + framework detection, nested-only-version (returns null), and
  quoted value.
- CHANGELOG.md — Unreleased entry.

## Diff summary
A single regex addition in the Dart branch of `detectStack`, parallel
to existing name-extraction. The `^` anchor in multiline mode keeps the
match column-zero so nested dependency `version:` lines are ignored.
The character class `[^'"\s#]+` covers the SemVer-plus-build-suffix
range that pubspec permits, while terminating on a `#` so inline
comments don't bleed into the captured value. No new dependencies, no
schema change (the `version` field already existed in the StackHit
type and was just always null for pubspec).

## Followups / blockers
