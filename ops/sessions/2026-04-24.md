## Picked (and why)
Notes detector's `firstMeaningfulLine` pulled an ATX heading body like
`**Status**: \`active\`` through unchanged, so the rendered summary
carried literal markdown syntax. `readme.ts` already has the same
concern and solves it with a local `stripMd` helper. I duplicated that
helper into `notes.ts` (per workspace AHA policy — three similar lines
beat a premature abstraction) and wired it into the return path. Small,
cosmetic, no schema impact.

## Diff summary
`stripInlineMd` removes backticks, bold/italic emphasis, underscore
emphasis, and markdown link wrappers — the same five patterns
`readme.ts::stripMd` already strips, intentionally duplicated rather
than extracted to a shared module. The change sits after the heading
regex, so ATX heading bodies (`# …`) and plain lines get the same
treatment. Length truncation still happens afterward on the stripped
text, which is the right order: truncate what the user will see, not
the raw markdown.
