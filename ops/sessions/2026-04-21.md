## Picked (and why)
Added `[[bin]]` detection to the entrypoints detector. `[[bin]]` is a
TOML array-of-tables, so a `[section]`-keyed reader like the one used
for `pyproject.toml` doesn't fit. Wrote a small dedicated scanner that
walks each `[[bin]]` header, slices the body up to the next section,
and extracts `name = "..."` (stripping `#` comments). Each surfaces as
`cargo run --bin <name>` with `command: null` — we deliberately do not
try to synthesize a command from the `path` field since the invoke is
what readers actually need.

## Diff summary
The Cargo scanner is ~30 lines and self-contained — it doesn't share
the pyproject section reader because array-of-tables semantics differ
(multiple headers with the same name, each producing a distinct
entry). The header regex uses `[ \t]*(?:\r?\n|$)` rather than `\s*` to
avoid swallowing the first body line on non-backtracking engines (not
strictly needed for V8 but it made the intent clearer after a
debugging detour on the first draft). Reviewer should confirm that a
`Cargo.toml` with no `[[bin]]` tables still returns `null` from
`detectEntrypoints` — covered by the new test.
