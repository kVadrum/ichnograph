## Picked (and why)
Julia detection. The last seven sessions all extended stack manifest
parsing (Dart, Gleam, Elixir, Ruby, and earlier deno/PHP), so the
groove is well-worn and the risk surface is small. Julia is a real
ecosystem with a clean canonical manifest, and adding it brings the
matrix to eleven languages without touching renderer code (the report
shape generically displays `language` / `name` / `version`). Followed
the gleam.toml scoping precedent — top-of-file region only — so a
`version = "..."` under `[compat]` (Julia's stdlib idiom for pinning
the Julia version itself) doesn't get mistaken for the package version.

## Diff summary
Adds a Julia detector after the Ruby block (and before composer.json)
that fires on either `Project.toml` or `JuliaProject.toml`. Name and
version are matched at the top of the file only (via the same
`split(/^\[/m)[0]` trick gleam.toml uses), so a `[compat]`/`[deps]`
subtable can't shadow package metadata. When both files exist,
`JuliaProject.toml` is preferred — that matches Julia's own load
order, where the `Julia`-prefixed file exists specifically to override
a generic `Project.toml`. No new dependencies, no schema change, no
renderer change.
