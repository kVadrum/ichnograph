## Picked (and why)
Fixed the Cargo `[[bin]]` header regex to tolerate trailing TOML comments
and whitespace, matching what a real TOML parser accepts. This is a
silent-failure class bug: a valid `Cargo.toml` with `[[bin]]  # cli`
would produce no command entries at all, with no warning. Narrow,
testable, and the cost of getting it wrong in the field is a confused
user who assumes ichnograph doesn't understand their project.

## Diff summary
The header regex for Cargo `[[bin]]` array-of-tables previously required
the line to end immediately after optional whitespace. TOML permits
comments after section headers, so any valid `Cargo.toml` that used
`[[bin]]  # comment` was silently skipped. The fix inserts an optional
`#[^\n]*` group before the line-terminator alternation. Behavior is
unchanged for all previously-matching inputs; a new test pins both the
comment case and the trailing-whitespace case.
