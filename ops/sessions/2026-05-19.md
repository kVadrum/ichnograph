## Considered
- Ruby `.gemspec` name/version reading — Gemfile-only detection currently
  returns `name: null, version: null`; .gemspec is the canonical identity
  artifact for Ruby gems.
- Cargo workspace awareness — `[workspace]`-only `Cargo.toml` files report
  null name/version; flagged in CLAUDE.md "Open questions". Larger surface
  than a daily slice (needs members enumeration design).
- Pubspec name with quoted YAML strings — narrow edge case; current `\S+`
  match works for the lowercase-identifier names Dart actually requires.

## Picked (and why)
Ruby `.gemspec` reading. Natural continuation of the recent
"read name/version from manifest" series (Elixir, Gleam, Dart, Deno),
fills a real gap, and fits the 15-minute slot. The Bundler-scaffolded
`spec.version = MyGem::VERSION` pattern is canonical for published gems,
so I also resolved that constant reference through to `lib/<gem>/version.rb`
— same shape as the `@version` resolution already in place for `mix.exs`.

## Changes
- `src/detect/stack.ts` — find first `*.gemspec` at root; read literal
  `spec.name` / `spec.version`; if version is a bare `CONST` reference,
  resolve via `lib/version.rb` or `lib/<dir>/version.rb`. Ruby is now
  detected from a gemspec alone (no Gemfile required); when both are
  present, gemspec wins as `manifest`.
- `tests/stack.test.ts` — four new cases: literal version, constant
  reference + `version.rb` resolution, gemspec-alone detection,
  Gemfile-alone preserved with manifest='Gemfile'.
- `CHANGELOG.md` — unreleased entry.

## Diff summary
Adds Ruby gemspec parsing to the stack detector. Single-quote and
double-quote literals are both accepted (existing pattern). The
VERSION-constant lookup is bounded to `lib/version.rb` plus
`lib/*/version.rb` (one level deep, no recursive scan), keeping cost
predictable. Anchored `^\s*` patterns avoid grabbing values from
commented-out lines. When a project has Gemfile only (no gemspec),
behavior is unchanged: Ruby detected, name/version null.

## Followups / blockers
- Cargo workspace handling (`[workspace]`-only Cargo.toml) remains an
  open item in CLAUDE.md — worth a dedicated session once the design
  is clear (do we list members? aggregate? pick first?).
- A `.gemspec` whose version literal is built from interpolation
  (`spec.version = "#{MAJOR}.#{MINOR}"`) still returns null. Vanishingly
  rare in practice; not worth the parser complexity.
