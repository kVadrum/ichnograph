## Considered
- Strip common inline HTML tags from readme/notes summaries —
  README first paragraphs frequently contain `<a href>`, `<br>`,
  `<p align="center">`, `<sub>`, `<kbd>`, etc., and they currently
  surface raw.
- Generic `<\/?[a-zA-Z]…>` HTML strip — too aggressive: would have
  broken the existing "leaves bare angle-bracketed words alone"
  contract (placeholders like `<name>`, `<your-token>`).
- Adding `pyproject.toml` / `Cargo.toml` entry-point detection —
  already shipped (v0.3.4 and v0.4.0); CLAUDE.md "Open questions"
  section is stale on that point.
- Whitespace collapse after tag strip — punted; tests use spacing
  that doesn't trigger double-space and the rule is small enough
  that future polish can layer on if needed.

## Picked (and why)
HTML tag strip with a curated allowlist. The allowlist (a, abbr,
b, br, cite, code, del, div, em, h1-h6, hr, i, img, ins, kbd, li,
mark, ol, p, pre, q, s, samp, small, span, strong, sub, sup,
table, tbody, td, th, thead, tr, u, ul, var) covers the
overwhelming majority of inline HTML found in READMEs while
keeping prose placeholders intact. This preserves the explicit
prior decision that bare `<word>` literals stay literal — the
tests for that contract still pass — and gives us a strict
improvement on the noisy cases without changing semantics for
ambiguous ones.

## Diff summary
The strip helper in both readme.ts and notes.ts gains one regex
that matches an opening or closing tag from a curated HTML tag
allowlist with optional attributes, replacing it with empty.
Order matters: it runs after the CommonMark autolink rules, so
`<https://x>` is consumed as an autolink first and never reaches
the HTML rule. The allowlist excludes anything that isn't a real
HTML tag, so `<name>` / `<your-token>` / `<unknown-flag>` placeholders
are still surfaced verbatim — both detectors have explicit tests
asserting that. A reviewer should mainly sanity-check the regex
boundary: the allowlist is a non-capturing group followed by `\b`,
ensuring `<abbr>` matches but `<abbreviation>` (not in list) does
not get half-stripped.
