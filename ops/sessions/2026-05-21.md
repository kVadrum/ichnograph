## Picked (and why)
Fixed the Dart pubspec `name` regex so it mirrors the existing
`version` regex: optional surrounding quotes are stripped and trailing
`#` comments terminate the value cleanly. Narrow, well-scoped, fits
the recent run of stack-detector polish, and surfaces a real difference
when YAML authors quote the package name. Added two tests for the
edge cases.

## Diff summary
The Dart pubspec name lookup previously used `^name:\s*(\S+)`, which
captured everything up to the next whitespace — including literal
surrounding quotes. The version lookup at the same site was already
quote-aware; this commit brings name to parity using the identical
regex shape. Behavior is unchanged for the common unquoted form like
`name: my_pkg`; the new behavior only affects quoted names and names
followed by a trailing `#` comment. No schema change, no new exports.
