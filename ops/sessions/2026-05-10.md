## Picked (and why)
PHP composer.json was the largest gap in the stack detector relative
to JS/Python: `frameworks: []` was hard-coded and `version` was always
`null`. The fix mirrors `detectJsFrameworks` exactly — walk a
`require` + `require-dev` merge against a label map — so there is no
new pattern to learn. Curated 15 entries covering the real-world
landscape (Laravel/Symfony/Drupal as the heavyweights, Filament and
Livewire/Inertia as the modern Laravel-stack additions, PHPUnit/Pest
for the test runners). `version` simply uses the existing
`stringOrNull` helper. Pure parity work, zero new runtime deps.

## Diff summary
The composer.json branch in `detectStack` now reads `version` (was
hard-coded `null`) and runs a `detectPhpFrameworks` helper symmetric
to the JS one: it merges `require` + `require-dev` and walks a fixed
label map. The map deduplicates labels so projects that pull both
`symfony/framework-bundle` and `symfony/symfony` only surface
`Symfony` once. Reviewer should confirm the label map matches house
style (entries are alphabetized within natural groups: full-stack
frameworks first, then microframeworks, CMS, runtime add-ons, test
runners). No render or schema changes — `frameworks` and `version`
were already in `StackHit`.
