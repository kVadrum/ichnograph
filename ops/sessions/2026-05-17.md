## Picked (and why)
Gleam version. Smallest, most mechanical of the candidates and directly
matches the shape of the last several sessions (deno.json, pubspec).
`gleam.toml` is plain TOML with `name` and `version` at top level, so the
regex pattern already used for `name` extends to `version` verbatim. The
one subtlety: a `version = "..."` line could appear under a
`[dependencies.foo]` subtable, so I scoped both name and version lookup
to the pre-section region of the file (split on the first `^[` header).
Existing name behavior is unchanged in the common case (top-of-file
name), strictly safer in the pathological case.

## Diff summary
Adds a top-level `version` regex match to the Gleam branch of
`detectStack` and scopes both name and version lookups to the region of
`gleam.toml` before the first `[section]` header. The scoping is a
defense against a `version = "..."` line inside `[dependencies.foo]`
being mistaken for the package's own version. For typical gleam.toml
files (name and version at the top, dependencies in their own table),
behavior is unchanged; the only observable difference is that Gleam
packages now report their version instead of `null`. Reviewer can spot-
check the `text.split(/^\[/m)[0] ?? text` idiom — it returns everything
before the first line starting with `[`, which is exactly the
top-of-file region we want.
