## Picked (and why)
Haskell support is a clean addition that follows the established pattern
(Ruby gemspec, Julia Project.toml, Gleam, Dart, Elixir). `.cabal` files
are line-oriented with column-0 fields, which makes the parse cheap and
robust. The main subtlety is that cabal stanzas (`library`, `executable`)
indent their own fields by convention — so a regex anchored at column 0
naturally skips them without needing explicit stanza tracking, and the
test pins that behavior.

## Diff summary
Adds a Haskell detector to `detectStack()` that fires when a root-level
`*.cabal` file is present. Name and version are pulled with case-
insensitive regexes anchored at column 0, so values nested inside
indented `library`/`executable` stanzas are not mistaken for the
package's own metadata. When more than one `*.cabal` file is present
(rare — cabal expects exactly one per package root), alphabetical sort
provides a deterministic tie-break. `package.yaml` (hpack) is
intentionally not consulted: hpack generates the `.cabal`, so the
`.cabal` already carries the resolved values.
