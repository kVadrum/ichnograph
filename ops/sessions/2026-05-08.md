## Picked (and why)
List-marker / task-checkbox stripping in the notes detector. STATE
and TODO files are exactly the surfaces this detector targets, and
they routinely lead with bullets rather than a `# heading`. The
fix is one focused branch in `firstMeaningfulLine`, parallel to
the existing heading-prefix match, with an explicit allowlist that
keeps prose dashes (`-foo`), bracketed shortcuts (`- [draft] note`),
and bare hyphens-without-trailing-space safe.

## Diff summary
The notes detector previously called `stripInlineMd` on the raw
trimmed line (or, for a heading line, on the heading body). This
adds an intermediate step: when the line is not a heading, try to
match a list-marker prefix (`-`/`*`/`+` or `\d+\.`) followed by an
optional GFM task checkbox (`[ ]`/`[x]`/`[X]`), and use the body
after that prefix as the input to stripInlineMd. The marker regex
requires trailing whitespace per CommonMark so prose like `-foo`
isn't matched, and the checkbox group is restricted to the GFM
forms so bracketed shortcuts like `[draft]` keep their literal
shape — the same conservative posture applied to reference-link
shortcuts in stripInlineMd. Reviewer should sanity-check the
restriction-to-non-heading-only branch (a heading containing a
list-shaped fragment, e.g. `# - WIP`, intentionally still surfaces
the dash).
