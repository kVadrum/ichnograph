## Picked (and why)
Fixed the multi-line HTML comment gap in both `notes.ts` and `readme.ts`.
Real-world READMEs and STATE files commonly carry a multi-line
copyright/license header at the top; without this, ichnograph surfaced
`<!--` as the title or summary. Pattern mirrors the existing fenced
code block skip — at start of the scan, digest the block; mid-paragraph,
terminate. Three places needed the same handling: notes' first-line
scan, readme's title scan, readme's paragraph scan.

## Diff summary
Each scan loop now treats a line that opens `<!--` without closing
`-->` on the same line as the start of a multi-line comment block,
advances `i` until the closing `-->` is found, and continues. The
guard is intentionally narrow (`startsWith('<!--')` + missing `-->`
on that line) so single-line comments still flow through the existing
strip helper. Reviewer should sanity-check that the `i++` cursor lands
where the surrounding for-loop's own `i++` expects — same shape as
the established fence skip, so identical to verified code.
