## Picked (and why)
The justfile recipe header regex required `\s*:` directly after the
recipe name, so any recipe with parameters (`build target='debug':`,
`bench *args:`, `deploy env stage:`) was silently dropped. Modern
justfiles lean on parameters heavily, so this is a real gap rather than
a corner case. The fix branches the header regex by source: justfile
gets a looser pattern that allows arbitrary content between name and
trailing `:`, while Makefile keeps the strict pattern (Makefile targets
don't have inline parameter syntax, and a looser regex would mis-match
rare assignment lines like `KEY = "x:y"`). The `(?!=)` lookahead still
rejects assignment forms (`set x := …`, `alias t := test`).

## Diff summary
The header regex for `readMakeLikeTargets` is now selected per source.
For `justfile`, it permits an optional `[ \t]+[^\n]*?` between the
recipe name and the trailing `:`, which is what justfile parameter
syntax requires. Makefile keeps the prior strict form. The lazy middle
match plus the `(?!=)` colon lookahead still correctly skip
`set shell := …`, `alias b := build`, and `KEY := value` lines. Reviewer
should sanity-check that the lazy `[^\n]*?` doesn't regress on tricky
justfile constructs — the new test covers parameter defaults with
quoted strings, variadic `*args`, and multi-token recipes.
