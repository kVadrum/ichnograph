## Picked (and why)
ATX closing-hash stripping. Concrete CommonMark spec violation (§4.2),
narrow surface (one regex per detector), fits the long-running thread of
markdown-stripping fixes the prior sessions have been on, and produces a
visibly cleaner summary for any STATE/TODO/README that uses the symmetric
`## Title ##` style. The whitespace requirement before the closing run
keeps `# foo#` literal — that's not a heading marker per spec, and the
fix preserves it correctly.

## Diff summary
The fix is a single `\s+#+\s*$` regex applied after the existing ATX
heading match captures the body. Because the strip requires preceding
whitespace, the no-space case (`# foo#`) is left literal as the spec
requires — that case is covered by an explicit test in both detectors.
The strip only applies when the line was matched as a heading; bullet
lines and prose are unaffected.
