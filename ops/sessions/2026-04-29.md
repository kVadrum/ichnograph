## Picked (and why)
Fixed the single-underscore italic gap in both `stripInlineMd` (notes)
and `stripMd` (readme). The bold counterpart `__text__` was already
handled, so `_text_` not being handled was clearly an oversight rather
than a deliberate choice. Used a lookbehind/lookahead pair
(`(?<![A-Za-z0-9_])_..._(?![A-Za-z0-9_])`) so intraword underscores in
identifiers like `CODE_OF_CONDUCT` or `tool_v2` aren't mistaken for
emphasis — Node ≥20 supports lookbehind, which the project already
requires. Added one test per detector covering both the strip and the
intraword guard.
